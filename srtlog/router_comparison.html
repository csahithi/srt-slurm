üîë Key Insights

  • TTFT Disaster: SGLang's cache_aware routing creates 174s avg wait vs Dynamo's 40s. This is because SGLang queues requests waiting for specific workers with cached KV.
  • Prefill Throughput: Per-request prefill is 15.6x faster with Dynamo because it doesn't create head-of-line blocking waiting for cache hits.
  • Transfer Efficiency: Dynamo's Running/Transfer ratio of 2.92x vs SGLang's 1.80x shows Dynamo decodes more tokens per KV transfer - more efficient use of disaggregation.
  • Load Balance: Dynamo decode workers have 3.6x better load balance (0.5% vs 1.8% CV), meaning work is distributed more evenly.
  • ITL Tradeoff: SGLang has 12% better ITL once tokens start flowing - the cache-aware placement helps decode locality. But this is overwhelmed by TTFT penalty.
  • Goodput: Only 18 requests (0.15%) met SLA with SGLang vs 1,166 (9.7%) with Dynamo - 65x improvement.
